HIGH PRIORITY ISSUES
1. **Missing `await` on `prisma.$disconnect()` in `finally` block**  
   The `finally` clause does not await the promise returned by `prisma.$disconnect()`. This can cause the script to exit before the database connection is properly closed, potentially leaking connections and leaving database sessions hanging. If the seed is run in automated pipelines, repeated runs could exhaust connection pools.

2. **Destructive update of Discord IDs on re‑seed**  
   Both Diego’s and Sebastian’s `upsert` updates unconditionally set `discordId` to `process.env.DISCORD_USER_… ?? null`. If the environment variable is missing or becomes unset, running the seed again will **nullify** an already‑configured Discord ID. This silently breaks Discord approval buttons and permissions for the affected staff accounts, causing data loss in development/staging environments where the seed is re‑applied.

3. **Hardcoded default passwords and console exposure**  
   The passwords `change_me_diego_2024!` and `change_me_sebastian_2024!` are embedded in source code and printed to the console (stdout/stderr) during every run. Even with a production‑use warning, the passwords are visible in logs, CI outputs, or terminal histories. This is a security practice that should be avoided—credentials should be injected via environment variables and never printed.
