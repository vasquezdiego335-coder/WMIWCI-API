HIGH PRIORITY ISSUES

1. **No environment variable validation** – If `RESEND_API_KEY` is missing or is still the placeholder `'re_placeholder'`, the Resend client is initialized silently with a non-functional key. All email sending will fail, and no error is logged here. The worker that uses this client may not detect the misconfiguration until it attempts to send, leading to **silent notification loss** (no Pre‑Approval or Final Confirmation emails go out).  
   **Impact:** Core customer messaging (the only two allowed email templates) becomes dead.  
   **Remedy:** Validate the API key at startup and throw/log immediately if it’s missing or equals the placeholder.

2. **Hardcoded fallback for `EMAIL_FROM` and `EMAIL_REPLY_TO`** – The fallback addresses (`hello@moveitclearit.com`) may not be verified in Resend’s domain. If the environment variables are missing, the resulting `from` address will be rejected by Resend, again causing silent delivery failures.  
   **Impact:** Same as above – all transactional emails fail.  
   **Remedy:** Require explicit configuration or validate domain verification early.

MEDIUM PRIORITY ISSUES

1. **Placeholder API key as anti‑pattern** – The string `'re_placeholder'` is used as a fallback. This masks missing configuration and makes debugging harder. A better approach is to throw an error at startup for production builds.  
2. **Missing logging** – No log line indicates what API key or sender address is being used, making it impossible to confirm the correct configuration in logs.  
3. **No type safety or configuration schema** – The file reads `process.env` directly; there is no centralised configuration schema or runtime check. The project has `src/lib/env.ts` for validating environment, but `resend.ts` does not use it.

LOW PRIORITY ISSUES

1. **No error handling on client instantiation** – Although `new Resend(...)` is unlikely to throw for an invalid API key (it just stores the string), the pattern is fragile if the library changes.  
2. **Single export instance** – No lazy initialisation or ability to re‑initialise if the key changes at runtime (unlikely, but could be considered).
